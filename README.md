# Above The Line: Translanguager's Companion

An advanced tool for exploring texts. Generate detailed grammatical annotations for any text using the Gemini API. This app was created by Dominik Lukeš in Google AI Studio and is powered by the Gemini API. It is designed as a comprehensive suite of tools for students, educators, and language enthusiasts to explore texts and video transcripts in depth.

## Core Features

- **Detailed Text & Video Annotation:** Paste any text or a timecoded video transcript to generate line-by-line analysis. The app provides word-for-word translations, detailed grammatical information, and natural-sounding idiomatic translations for each line.
- **Text-to-Speech (TTS) Slideshow:** For annotations without a video source, the slideshow can generate high-quality audio for each slide, turning your text into an engaging listening experience with full playback controls.
- **Multiple Interaction Modes:** Go beyond static text with three dynamic modes designed for different learning styles: a focused **Deep Read** view, a multimedia **Slideshow** mode for syncing text with video, and an interactive **Study Mode** to gamify your learning.
- **Progress Tracking & Gamification:** Turn your annotations into practice sessions. Study Mode tracks your score, time, and progress, helping you identify areas for improvement.
- **Flexible Data Management:** Save your work to your browser, export everything for backup, or share with others. You can export individual annotations as JSON or PDF, and even export stanzas as PNG images.

## Architecture

This application is built as a client-side Single Page Application (SPA) using **React** and **TypeScript**. It leverages a serverless architecture where the "backend" logic is primarily handled by the **Google Gemini API**.

### Tech Stack

- **Framework:** React 19 (via Vite)
- **Language:** TypeScript
- **Styling:** Tailwind CSS with `@tailwindcss/typography`
- **State Management:** React Hooks (`useState`, `useReducer`) + Custom Hooks (`useLocalStorage`)
- **AI Integration:** Google GenAI SDK (`@google/genai`)

### Data Persistence Strategy

The application uses a hybrid local-first storage approach to ensure data privacy and offline capability without requiring a dedicated backend database:

1.  **LocalStorage:** Stores lightweight data such as user preferences (theme), text annotations (JSON), and study session history.
2.  **IndexedDB:** Used via a custom utility wrapper to store heavy binary data, specifically the **audio blobs** generated by the Text-to-Speech service. This avoids the 5MB quota limit of LocalStorage.

### AI Implementation

The application interacts with the Google Gemini API for several distinct tasks:

-   **Annotation Generation:** Uses `gemini-2.5-flash` with Structured Output (JSON Schema) to parse texts into stanzas, lines, words, and grammatical data.
-   **Title Generation:** Analyzes text content to generate concise, descriptive titles automatically.
-   **Gender Detection:** Analyzes character names and dialogue context to infer gender, allowing for appropriate voice selection in TTS.
-   **Text-to-Speech:** Uses `gemini-2.5-flash-preview-tts` to generate natural-sounding audio for individual text segments.

## Deployment

Since the application is a static site, it can be deployed to any static hosting provider (Vercel, Netlify, GitHub Pages, etc.).

### Prerequisites

-   Node.js (v18 or higher)
-   npm or yarn
-   A Google Cloud Project with the Gemini API enabled.

### Setup

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/techczech/Above-the-Line.git
    cd Above-the-Line
    ```

2.  **Install dependencies:**
    ```bash
    npm install
    ```

3.  **Environment Configuration:**
    Create a `.env` file in the root directory. You must provide your Google Gemini API key here.
    ```env
    API_KEY=your_google_gemini_api_key_here
    ```

### Development

To start the local development server:

```bash
npm run dev
```

### Production Build

To build the application for production:

```bash
npm run build
```

The output will be generated in the `dist` directory, which can then be served by any static file server.
